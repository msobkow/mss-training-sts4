Welcome to Mark Stephen Sobkow's Training for Spring Tool Suite 4 (mss-training-sts4)!

All of Mark Stephen Sobkow's Open and Shared Source projects have their web presence hosted under https://msscf.org/ subdomains, with one subdomain per project. All of the git repositories for same are hosted at https://github.com/msobkow at this time, though as I move forward with incorporation, I will set up a corporate git account and transfer ownership of the Training for Spring Tool Suite 4 to that corporate entity.

This project comprises a family of implemented and planned architectural prototypes to flesh out the implementation details of the various layers I need for my plans for the MssCFe application I have in mind for future development. Most of the architectural layers I'll need are common to many Spring Boot applications, and I'm already finding a few of my planned prototypes getting bumped up the priority list to deal with client's needs that are more pressing than my own.

The main project pom for this family of prototypes contains a number of sub-module prototypes. Each sub-project package deals with a new architecural layer that needs to be prototyped so I know how to tie the pieces together in real world applications.

Note that the intent of this project is NOT to provide training to other internet users, but to serve as a playground for my own integration training efforts. I just thought that as I work through the details, others might find my up-to-date samples of approaches others have taken in the past to be useful refreshers for those original code samples and training guides. I certainly don't take credit for the bulk of the algorithms and approaches in my prototypes - they all leverage example code and documentation and help tips from the internet at large. When I run across a new source, I try to comment on the URL in the code, but that is as far as any credits to original thought go.

There is a separate page on the MssCFe Architectural Notes which describes the architecture I'm tryhing to prototype. Where there is disagreement between that document and this document as to the prototype modules and their development order and this document, the MssCFe document takes precedence.

Architectural Prototypes Implemented So Far

org.msscf.sts4training.config

This rather simple prototype deals with integrating Spring Boot applications with configuration property files that can be overridden at the command line in the usual Java CLI fashion.

org.msscf.sts4training.javafx

This challenging-to-write prototype integrates JavaFX 14 with OpenJDK 14 and Spring Boot using the code provided by Mario Jauvin as the starting point. More details are on the sub-page org.msscf.sts4training.javafx.

Architectural Prototypes In Progress

The Web Document Service will provide read-only access for downloading a client installation's subscribed documents in order to be able to do things like manufacture code. This service is intended to be generic; it just stores text BLOBs that specify a MIME type defining their text content type. All text BLOBs must be valid in the UTF-8 character set. Details are on the sub-page org.msscf.sts4training.webdocsrv.

Architectural Prototypes Planned For Near Future Due To Coinciding Client Needs

org.msscf.sts4training.cfbmr

The Code Forge Business Model Repository is the back-end data store for the models in a model repository server, which is intended to be deployed as a canned VM instance to subscribing clients. Details are on the sub-page org.msscf.sts4training.cfbmr.

All other MssCFe prototypes will be deferred until I have a corporation set up to own the work instead of me owning it personally.

Down the road: Web UIs

That should pretty much cover the architectural prototypes I'll need to cover the MssCFe, and a rather significant chunk of the typical bread and butter web stack.

What I'll have to tackle after that is a React web interface as that is the most common in-demand user interface technology the job market asks for.
